# 📄 iOS 智能 AI 相机 MVP 开发设计全手册
### 🛠 一、 项目核心定义
产品名称：AI Director Camera (暂定)  
核心场景：  
1.智拍模式（Coach Mode）：面向情侣/闺蜜，AI 扮演导演，通过大模型审美分析给出构图与姿态建议，并利用原生框架实现实时纠偏引导。  
2.抓拍模式（Capture Mode）：面向家长带娃，AI 扮演专业摄影师，通过预测动作意图与表情分析，自动捕捉“决定性瞬间”。
### 🏗 二、 技术架构方案
#### 1. 硬件资源分流 (Heterogeneous Computing)
* CPU: 处理 Swift 业务逻辑、状态机切换、UI 响应。  
* GPU (Metal): 负责相机预览流渲染、实时滤镜、以及 mlx-swift (大模型) 推理。  
* ANE (Apple Neural Engine): 负责 Vision 框架（骨骼/人脸识别）与 Core ML 模型（美学评分）。  
#### 2. 软件栈分层
* 采集层: AVFoundation (4K/30fps 或 1080p/60fps 采样)。
* 决策层 (大脑): mlx-swift 驱动的 Moondream2 或 Qwen2-VL (4-bit 量化)。频率：0.5Hz - 1Hz。
* 执行层 (小脑): Vision Framework (VNDetectHumanBodyPoseRequest, VNFaceObservation)。频率：15Hz - 24Hz。
* 逻辑层: 差值分析算法 (Difference Analysis)，将目标向量与实时向量对比并输出文案。
### ⚙️ 三、 核心算法底层逻辑
#### 1. 智拍模式：差值引导算法
* 输入：AI 指令 TargetPose(joint: .leftWrist, relation: .higherThan, ref: .nose)。
* 处理：获取 CurrentPose 中对应结点的 Normalized 坐标 (x, y)。
* 计算：DeltaY = Current.leftWrist.y - Current.nose.y。
* 输出：若 DeltaY < 0.1，触发 UI 震动并提示：“手再抬高一点点”。
#### 2. 抓拍模式：预测与评估逻辑
* 预缓冲：使用 AVCaptureVideoDataOutput 维护一个 30 帧的环形缓冲区。
评分权重：
* FaceQuality (0.4) + SmileProbability (0.3) + CompositionScore (0.3)。
* 触发：当总分连续 3 帧超过 0.85 阈值，立即提取缓冲区中评分最高的一帧进行高画质保存。
### 📝 四、 验证核心功能可行性
分成三个阶段开发核心功能  
#### 阶段 1：构建异步双路相机底层
当前这个项目是已经创建的空的Swift项目,在这个空的项目中编写一个 iOS Swift 项目的基础架构。要求：  
1. 使用 AVFoundation 创建一个自定义相机，实时预览显示在界面上。
2. 实现一个`双路采样策略`：
    * 路径 A (高频)：每秒 20 次将帧传给 Vision 框架处理人体骨骼识别，结果绑定到 ANE (Neural Engine) 执行。
    * 路径 B (低频)：每秒 1 次将帧缩放至 384x384 并转换为适合 MLX 推理的格式。
3. 使用 Swift 并发 (async/await) 确保低频路径不阻塞高频预览。
4. 界面底部预留一个状态栏，用于显示‘导演建议’文字。"
5. 权限声明：Info.plist 必须包含 `NSCameraUsageDescription`、`NSMicrophoneUsageDescription` 和 `NSPhotoLibraryAddUsageDescription`。
#### 阶段 2：实现姿态差值引导逻辑
基于之前的代码，请实现一个`智拍引导引擎`：  
1. 定义一个结构体 PoseConstraint，包含目标关节、参考点、以及位置关系（如高于、左于、夹角等）。
2. 编写一个函数，接收 Vision 框架返回的 VNHumanBodyPoseObservation，并与 PoseConstraint 对比。
3. 逻辑示例：如果目标要求`左手高于头顶`，但实际坐标不符，请在 UI 界面中心显示一个动态箭头和文字‘请抬高左手’。
4. 要求使用平滑算法（如移动平均滤波），防止由于检测波动导致的引导文案闪烁。"
#### 阶段 3：实现智能抓拍逻辑
请为相机添加‘智能抓拍模式’功能：
1. 使用 Vision 框架的 VNDetectFaceCaptureQualityRequest 实时监控人脸拍摄质量。
2. 结合 VNFaceObservation 中的 hasSmile 属性。
3. 实现一个简单的评分器：Score = (CaptureQuality * 0.6) + (SmileProbability * 0.4)。
4. 当 Score 超过 0.8 时，自动调用 AVCapturePhotoOutput 拍照，并使用 Taptic Engine 给予用户震动反馈，同时将照片保存至相册。"
#### 其他要求
1. 热量管理：在 videoDataOutput 中添加逻辑，若 ProcessInfo.thermalState 为 .serious 或 .critical，强制将 Vision 频率降至 5fps。
2. 项目中已经导入了必要的xml-swift第三方库,MLX 推理过程需要使用`mlx-community/Qwen2.5-VL-3B-Instruct-4bit`模型,对于的API枚举类型是`VLMRegistry.qwen2_5VL3BInstruct4Bit`
3. 内存限制：MLX 推理时，务必在推理完成后显式释放不必要的 Tensor 缓存，防止内存溢出导致 App 崩溃。
